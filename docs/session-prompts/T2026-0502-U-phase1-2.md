**【効果検証 Step 1】**: misses が 0 でない場合、それは「高信頼な間違い」を意味する → Phase 2 進めると本番で false merge が起きるリスク。
- misses ≥ 1: 閾値 (`COSINE_HIGH=0.85` / `COSINE_LOW=0.65`) を embedding_bench.py の出力 "推奨単一閾値" を見て調整 → 再 bench
- それでも miss が消えない: voyage-3-lite API モデルに切替検討（embedding_judge.py に `VoyageEmbeddingBackend` を追加実装）
- どうしても通らない: T2026-0502-U Phase 2 を中止して TASKS.md に「embedding 移行不採用 (理由: ...)」を記録して close

```python
from config import ANTHROPIC_API_KEY as _ANTHROPIC_API_KEY  # noqa: E402
_AI_MERGE_ENABLED = os.environ.get('AI_MERGE_ENABLED', 'false').lower() == 'true'
_EMBEDDING_MERGE_ENABLED = os.environ.get('EMBEDDING_MERGE_ENABLED', 'false').lower() == 'true'

_ai_judge = None
if _EMBEDDING_MERGE_ENABLED:
    # T2026-0502-U Phase 2: embedding ベース (月$0)
    try:
        from embedding_judge import make_default_judge
        _ai_judge = make_default_judge(api_key=_ANTHROPIC_API_KEY)
        if _ai_judge:
            print('[merge_judge] embedding judge enabled (T2026-0502-U)')
    except Exception as e:
        print(f'[merge_judge] embedding judge 初期化失敗 → fallback: {e}')

if _ai_judge is None and _ANTHROPIC_API_KEY and _AI_MERGE_ENABLED:
    # Legacy: Haiku merge (cost $120/month・要 AI_MERGE_ENABLED=true)
    _ai_judge = AIMergeJudge(api_key=_ANTHROPIC_API_KEY)
    print('[merge_judge] Haiku judge enabled (legacy)')

if _ai_judge is None:
    print('[merge_judge] disabled (env: EMBEDDING_MERGE_ENABLED=false, AI_MERGE_ENABLED=false)')
```
