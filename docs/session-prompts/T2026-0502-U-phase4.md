## あなたへの依頼（要約）

Phase 2-3 で fetcher / processor の両方が embedding を使うようになった結果、
**毎 fetcher run で 5000 topics × DynamoDB Read が走る** 状態になる。
DynamoDB RCU コストは小さい ($1.80/月) が、レイテンシ削減と将来の embedding 拡張
（より多次元・より多 topic）に備えて、embedding を **S3 集約 JSON にキャッシュ**する
最適化を入れる。「やらなくても支障ないが、やると未来が綺麗」枠の改修。

## 背景（必読）

### 現状（Phase 3 完了時点）

```
fetcher Lambda (毎 30 分):
  → 既存 5000 topics の embedding を DynamoDB から read
  → 新クラスタ embedding 計算
  → 類似度判定
  → DynamoDB write 数件 (新マージ判定結果)

processor Lambda (1 日 2 回 + on-demand):
  → 各 topic の latestKeyPointEmbedding を DynamoDB から read
  → 新 articles の embedding 計算
  → skip 判定
```

### 提案する最適化

```
processor Lambda (1 日 2 回):
  → AI 生成完了後、全 topic の latestKeyPointEmbedding を集約して
    s3://p003-news-XXX/embeddings/topics-embedding.json として 1 回 PUT
    (~2.5MB / qint8 384 次元 / 5000 topics)

fetcher Lambda (毎 30 分):
  → 起動時に s3://.../topics-embedding.json を 1 回 GET (~50ms)
  → メモリ上で類似度計算
  → DynamoDB read 不要

S3 GET 数:
  - 48 fetcher run/日 × 30 = 1,440 GET/月
  - $0.0004/1000 req × 1.44 = $0.0006/月（誤差）

DynamoDB read 削減:
  - 既存: 240K read/日 × 30 = 7.2M read/月 → $1.80/月
  - 新: 0 (fetcher 経由は S3)・processor の write 系のみ残る
  - 削減効果: ~$1.80/月 → ~$0/月
```

**コスト削減効果は誤差** だが、**レイテンシ短縮** (DynamoDB BatchGet 5000 items vs S3 GetObject 1 file) と **将来の拡張性** (embedding を 768 次元・10000 topics に増やしても線形に増えない構造) が本質的価値。

### Step 1: embedding_cache_writer.py 新設（10 分）

```python
# lambda/processor/embedding_cache_writer.py
import base64
import json
import boto3

S3_BUCKET = os.environ.get('S3_BUCKET', '')
CACHE_KEY = 'embeddings/topics-embedding.json'


class EmbeddingCacheWriter:
    """processor が AI 生成完了後に全 topic embedding を S3 に集約 PUT する。"""

    def __init__(self):
        self.s3 = boto3.client('s3')

    def write_all(self, topics: list[dict]) -> int:
        """topics の latestKeyPointEmbedding を全部集約して S3 PUT。

        Args:
            topics: list of {'topicId': str, 'latestKeyPointEmbedding': List[float]}
        Returns:
            書き込んだ topic 数
        """
        payload = {
            'updatedAt': datetime.now(timezone.utc).isoformat(),
            'embeddings': {
                t['topicId']: t['latestKeyPointEmbedding']
                for t in topics
                if t.get('latestKeyPointEmbedding')
            },
        }
        body = json.dumps(payload).encode('utf-8')
        self.s3.put_object(Bucket=S3_BUCKET, Key=CACHE_KEY, Body=body,
                           ContentType='application/json')
        return len(payload['embeddings'])
```

### Step 2: embedding_cache_reader.py 新設（10 分）

```python
# lambda/fetcher/embedding_cache_reader.py
import json
import time
import boto3

S3_BUCKET = os.environ.get('S3_BUCKET', '')
CACHE_KEY = 'embeddings/topics-embedding.json'
MAX_AGE_SECONDS = 86400  # 24h より古かったら無効扱い


class EmbeddingCacheReader:
    """fetcher が起動時に S3 から embedding 集約 JSON を read してメモリ展開。"""

    def __init__(self):
        self.s3 = boto3.client('s3')
        self.embeddings: dict[str, list[float]] = {}
        self.loaded_at = None

    def load(self) -> int:
        try:
            obj = self.s3.get_object(Bucket=S3_BUCKET, Key=CACHE_KEY)
            payload = json.loads(obj['Body'].read())
            updated_at = payload.get('updatedAt')
            self.embeddings = payload.get('embeddings', {})
            self.loaded_at = time.time()
            print(f'[embedding_cache] loaded {len(self.embeddings)} topics from s3 (updated={updated_at})')
            return len(self.embeddings)
        except self.s3.exceptions.NoSuchKey:
            print('[embedding_cache] no cache yet (first run)')
            return 0
        except Exception as e:
            print(f'[embedding_cache] load failed: {e} → fallback to DynamoDB')
            return 0

    def get(self, topic_id: str) -> list[float] | None:
        return self.embeddings.get(topic_id)
```
