### skip 判定が有効な条件

processor は `pendingAI=True` の topic を順に処理する。実態:
- 既に `keyPoint` がある topic でも、新 article 1 件追加で全文再生成している
- 新 article のタイトル / 本文 が既存 keyPoint と意味的に重複している場合、
  AI 再生成しても結果はほぼ同じ（無駄）

### 判定アルゴリズム

```
for topic in pending:
    new_articles = get_new_articles_since(topic.lastProcessedAt)
    if not new_articles:
        skip (no work)
        continue
    
    # 既存 keyPoint の embedding (永続化済) を読む
    existing_kp_emb = topic.get('latestKeyPointEmbedding')
    if existing_kp_emb is None:
        # 初回処理 = embedding 未保存 = 通常通り AI 生成
        proceed_with_ai_generation()
        continue
    
    # 新 articles のタイトル群を encode
    new_titles_embs = embedding_backend.encode([a.title for a in new_articles])
    
    # 各新 article が既存 keyPoint と類似してるか
    similarities = [cosine(emb, existing_kp_emb) for emb in new_titles_embs]
    
    if max(similarities) > SKIP_THRESHOLD (0.95):
        # 全部既存 keyPoint と高類似 = AI 再生成スキップ
        topic['lastProcessedAt'] = now()  # touch
        topic['skippedByEmbedding'] = True  # 観察フラグ
        # keyPoint / perspectives / outlook はそのまま
        continue
    
    # 1 件でも sim < 0.95 = 新情報あり = AI 生成
    proceed_with_ai_generation()
    # AI 生成後に新 keyPoint embedding を保存
    topic['latestKeyPointEmbedding'] = embedding_backend.encode([new_keypoint])[0]
```

### Step 1: embedding_skip.py 新設（30 分）

**設計方針**:
- `lambda/fetcher/embedding_judge.py` の `OnnxEmbeddingBackend` を import 流用
- Lambda layer `p003-embedding-e5small` を processor Lambda にも attach（Phase 2 で fetcher のみだったので）
- 環境変数 `EMBEDDING_SKIP_ENABLED` で disable 可（kill switch）
- skip 判定の閾値 `EMBEDDING_SKIP_THRESHOLD=0.95`（env でオーバーライド可）

```python
# lambda/processor/embedding_skip.py
import os
import sys

# fetcher 側のモジュールを参照（Lambda layer 上は同じパスにマウント）
sys.path.insert(0, os.path.dirname(os.path.dirname(os.path.abspath(__file__))) + '/fetcher')
from embedding_judge import OnnxEmbeddingBackend

SKIP_THRESHOLD = float(os.environ.get('EMBEDDING_SKIP_THRESHOLD', '0.95'))


class EmbeddingSkipJudge:
    """新 articles と既存 keyPoint の意味類似度で AI 再生成 skip を判定する。"""

    def __init__(self, backend=None):
        self.backend = backend or OnnxEmbeddingBackend()
        self.skipped_count = 0
        self.proceeded_count = 0

    def should_skip(self, existing_kp_emb, new_article_titles):
        """既存 keyPoint embedding と新 articles タイトル群を比較。

        Returns:
            (skip: bool, max_sim: float)
        """
        if existing_kp_emb is None or not new_article_titles:
            return False, 0.0
        new_embs = self.backend.encode(new_article_titles)
        sims = [self.backend.cosine(existing_kp_emb, e) for e in new_embs]
        max_sim = max(sims)
        skip = max_sim > SKIP_THRESHOLD
        if skip:
            self.skipped_count += 1
        else:
            self.proceeded_count += 1
        return skip, max_sim

    def encode_keypoint(self, keypoint_text):
        """生成された keyPoint を embedding 化（保存用）。"""
        if not keypoint_text:
            return None
        return self.backend.encode([keypoint_text])[0]
```

    new_articles = ...articles since lastProcessedAt...
    
    if _skip_judge and topic.get('latestKeyPointEmbedding') and new_articles:
        skip, max_sim = _skip_judge.should_skip(
            topic['latestKeyPointEmbedding'],
            [a.get('title','') for a in new_articles]
        )
        if skip:
            print(f'[skip] {topic_id}: max_sim={max_sim:.3f} > 0.95 → AI 再生成スキップ')
            topic['skippedByEmbedding'] = True
            topic['lastProcessedAt'] = now()

            continue
    
    # 通常の AI 生成パスへ
    new_story = generate_story(...)

    if _skip_judge and new_kp_text:
        topic['latestKeyPointEmbedding'] = _skip_judge.encode_keypoint(new_kp_text)
    save(topic)
