# Flotopic プロダクト方針（2026-04-28 更新）

## AI生成物の構造（確定）

whyNowフィールドは廃止。AIの生成物は以下の4要素のみ：

1. **状況解説** — 何が起きているか。「もともと○○という状況があって、△△が起き、今□□の段階にある」という時系列ストーリー形式。箇条書き禁止。簡潔でキレのある日本語で書く。
2. **各社の見解** — 主要プレイヤーの立場・動き
3. **これからの注目ポイント** — 「AI予想」という表現は使わない。誠実な言い換え。
4. **予想判定** — 後から新記事が来たとき、当否を自動判定

## 2軸の明確な分離

- **注目度軸** = グラフ・記事数・velocityScore がデータで示す。AIが「なぜ今重要か」を語らない。鮮度が命。
- **内容軸** = ストーリー解説・各社見解・注目ポイント。AIが語る。グラフの鮮度とは独立。

keyPoint（注目ポイント）は「ストーリー内容の中での注目すべき点」。velocityScore（みんなの注目度）とは別物。

## 状況解説のプロンプト方針

「簡潔でキレのある日本語で、時系列のストーリーとして書け。情報を並べるのではなく、言葉を選んで格好よく語れ。背景→変化→現在の状況 の流れで。」

## 後回しにしているもの

- 自分ごと感（personalImpact/stakeholders）— 現在の4要素枠には含まれない。別途検討。
- ストーリー分岐のセマンティック判断（T2026-0428-BRANCH）

## 配信頻度の方針（2026-05-02 確定 = 朝刊・夕刊モデル）

**Flotopic は「朝刊・夕刊が配信される news サイト」として設計されている。**

- processor (AI 要約生成) は **2x/日 cron** (5:30 / 17:30 JST) のみで稼働
- fetcher → processor の即時 invoke は撤廃済（T2026-0502-J / 2026-05-02・PR #146）
- 新トピックの初動 UX は **平均 6h・最悪 12h は AI 要約ナシ**（RSS スニペットのみ表示）の状態を許容
- これは想定された UX trade-off であり、コスト（Anthropic API +$17/月）と品質サイクル運用のバランス点

**やってはいけないこと**:
- レイテンシ短縮を理由に fetcher → processor 即時 invoke を復活させること（コスト 67% 増・PO 判断で却下済）
- 上記なしで cron 頻度を増やすこと（コスト垂れ流しに直結）

**復旧パスが必要になった場合**:
- cron 2x/日 → 4x/日に増やす（朝刊・夕刊 → 4 版体制）。コストは線形に増えるが流出は予測可能
- 即時性が決定的に必要な特定トピックのみ別 path で処理する設計（要 PO 判断）

**要約品質の改善**は配信頻度ではなく **プロンプト改善サイクル** (T2026-0501-K 等のフェーズ 2 タスク) で回す。配信頻度と要約品質は独立軸。

## フェーズ前提を壊さない原則（2026-05-02 制定 / T2026-0502-AQ 由来）

**フェーズ N の前提を壊す可能性のある変更は、PR description に `Phase-Impact-Conflict:` 行で明示する。**

由来: T2026-0502-ADSENSE-FIX (4/28) で AdSense 薄コンテンツ対策として `frontend/topic.html` を `noindex` 化した結果、Cloudflare PV が 4/27 260 → 4/28 20 (1/10) に急減し 5 日間誰も気づかなかった。フェーズ4 (収益化) 系の局所最適化が、フェーズ3 (UX/成長) の前提 (= SEO 流入経路) を壊した事例。

**特に注意すべきフェーズ前提**:
- **フェーズ3 の前提 = 流入経路**: SEO・SNS シェア・直接訪問・登録ユーザーの再訪。これらを縮小する変更 (noindex 化・URL 変更・robots.txt 制限・OGP タグ削除) は Phase-Impact-Conflict 必須
- **フェーズ2 の前提 = AI 生成パイプライン**: fetcher / processor の cron 頻度・プロンプト・スキーマ。これらを変える PR は keyPoint 充填率や perspectives 充填率への影響を述べる
- **フェーズ1 の前提 = 観測点**: SLI / freshness-check / cf-analytics. これらの SLI 計算式や閾値を緩める変更は、なぜ緩めるかを明示する

**Phase-Impact-Conflict 記載例**:
```
Phase-Impact-Conflict: フェーズ3 (UX/成長) — topic.html を noindex 化することで
SEO 流入が消失する可能性。Cloudflare PV の前日比 50% 落ちが想定される。
許容根拠: AdSense 通過のため一時的に SEO を犠牲にする。AdSense 通過後 (Eval-Due 2026-05-09)
直ちに noindex を外す PR を出す。
```

**物理化候補** (1 ヶ月運用後検討): PR template への `Phase-Impact-Conflict:` 必須行追加、`commit-msg` hook での grep 強制。

## 長期ビジョン（2026-05-01 確定）

### 情報の地図
ユーザーが「今の世界でどこに何が起きているか」を俯瞰できるプラットフォーム。トピックは孤立して存在するのではなく、互いに接続し、地形図として機能する。velocityScore・storymap・関連トピックリンク・タイムラインがその構成要素。全ジャンルを揃えることが「地図の完全性」になる。ジャンルを絞ることはしない。

### 外部収益化（地図完成と並走）
地図の品質が上がり次第、SNS化より先に収益化を優先する。広告（忍者AdMax）のCPM向上・AdSense審査通過・アフィリエイトがターゲット。ユーザー数よりコンテンツ密度・滞在時間・ページ品質が先決。

### SNS化（収益化後・ユーザーが集まってから）
ユーザーがその地図の上を動き回りながら、お気に入り・閲覧履歴・反応が蓄積される。SmartNewsは「配達」、グノシーは「まとめ」、Flotopicは最終的に「広場」を目指す。ユーザー数が増えてから実装する。

**コメント機能**はSNS化フェーズの最初の実装。記事ではなく「トピック」にコメントがつくのがFlotopicならでは——「このニュース、あなたはどう読む？」という議論の場。コメントが存在すること自体が「人が集まっている証拠」になり、差別化と回帰動機の核心になる。

### 低コスト化の意味
現在の運用コスト極小化はゴールではなく、地図+収益化ビジョンを実現するまで生き延びるための投資。フェーズ1の基盤強化は全てこの長期視点での合理的判断。
