# Flotopic アナリティクスガイド

## 収集しているデータ

Flotopic では、ユーザー体験の改善とコンテンツ品質向上のために以下のイベントデータを収集しています。

- **ページビュー（view）**: トピックページが表示された回数。`topic_id`・タイムスタンプ・ユーザーの大まかな地域情報を記録します。
- **トピッククリック（click）**: トップページから各トピックへのクリック数。どの見出しが読者の興味を引いたかを把握します。
- **セッション時間（session）**: ページ滞在時間（秒）。記事の読了率を推定するために利用します。

個人を特定できる情報（IPアドレス・氏名・メールアドレスなど）は収集していません。

## flotopic-analytics テーブルの読み方

DynamoDB テーブル名: `flotopic-analytics`（リージョン: `ap-northeast-1`）

| 属性名 | 型 | 説明 |
|---|---|---|
| `event_id` | String（PK） | イベントの一意ID（UUID） |
| `timestamp` | Number（SK） | Unix タイムスタンプ（秒） |
| `event_type` | String | `view` / `click` / `session` |
| `topic_id` | String | 対象トピックのID |
| `title` | String | トピックタイトル（スナップショット） |
| `view_count` | Number | 集計済みビュー数（バッチ集計時に付与） |
| `duration_sec` | Number | 滞在時間（`session` イベントのみ） |

週次の編集レポートでは、過去7日間の `view` イベントを `topic_id` ごとに集計し、上位10件を「先週のトップトピック（PV順）」として表示します。

## 今後のロードマップ

将来的には以下の分析機能を追加予定です。

- **ヒートマップ**: トピックページ内のどのリンクがよくクリックされるかを可視化し、UI改善に活用する。
- **ファネル分析**: トップページ閲覧 → トピック閲覧 → 外部記事クリックの離脱率を計測し、コンテンツの訴求力を定量評価する。
- **A/Bテスト（タイトル最適化）**: AIが生成したトピックタイトルの複数バリエーションを試験配信し、クリック率の高いタイトル形式を学習する。
