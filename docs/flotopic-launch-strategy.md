# Flotopic 公開ロードマップ & 成長戦略

> CEOが自律的に進める戦略ドキュメント。フェーズ移行の判断基準を明記する。
> 条件を満たしたらCEOが自ら次フェーズへの移行提案を出す。

最終更新: 2026-04-22

## フェーズ定義と移行条件

### Phase 0 ── インフラ整備（現在）
**目標**: 安定稼働と基本品質の確保

完了条件:
- [x] flotopic.com ドメイン取得
- [x] CloudFront + SSL 設定（home PCで実行待ち）
- [x] AI要約・差分更新・重複排除稼働
- [x] 専門AI 7体稼働（git push後）
- [ ] https://flotopic.com でアクセス確認
- [ ] AI要約が全トピックに表示されている

**移行条件**: HTTPSアクセス確認 + AI要約正常動作

---

### Phase 1 ── オーガニック流入獲得（目標: 月1,000PV）
**開放機能**: 閲覧のみ（認証なし・コメントなし）

やること（CEOが毎週進める）:
- X（@flotopic_jp）自動投稿で認知獲得
- 週次まとめ記事（weekly_digest.py）でSEO流入
- RSS配信でニュースリーダーユーザーを獲得
- Google Search Consoleに登録、インデックス促進
- sitemap.xmlをGoogle/Bingに送信

KPI監視:
- Cloudflare Analytics で週次PVを確認（CEOが月曜に報告）
- 上位トピックのX拡散数（マーケティングAIが報告）

**移行条件**: 月間1,000PV 達成

---

### Phase 2 ── エンゲージメント強化（目標: 月5,000PV・リピーター獲得）
**開放機能**: Google ログイン + お気に入り機能

移行トリガー: 月1,000PV達成をCEOが検知 → ナオヤに移行提案 → 承認後に機能ON

やること:
- Google OAuth有効化（コード済み、config.jsでCLIENT_ID設定するだけ）
- お気に入り機能をLambda + DynamoDBで公開
- ユーザー行動データの蓄積開始（どのジャンルが人気かの分析）
- パーソナライズ（好みのジャンルを記憶）の強化

KPI:
- 月間ログインユーザー数
- お気に入り登録率（閲覧数に対する比率）
- リピート訪問率（Cloudflare Analyticsで確認）

**移行条件**: 月5,000PV + アクティブユーザー100人

---

### Phase 3 ── コミュニティ形成（目標: 月2万PV・DAU 500人）
**開放機能**: Googleログイン必須コメント + いいね機能

移行トリガー: Phase 2条件達成をCEOが検知 → セキュリティ確認 → 提案

やること:
- コメント機能の公開（Googleログイン必須、実装済み）
- セキュリティインフラ有効化（flotopic-rate-limits、flotopic-analytics）
- Lambda同時実行数制限設定（コスト攻撃対策）
- 荒らし対応フロー整備（Ban機能、NGワードリスト拡充）
- 編集AIによるコメントモデレーション強化

リスク管理:
- コメント公開前に必ずdeploy-security.shを実行
- 最初の1週間はCEOが毎日コメントをモニタリング
- 問題発生時はコメント機能を即座にOFF（config.jsのCOMMENTS_URLを空にするだけ）

**移行条件**: 月2万PV + セキュリティインフラ確認済み

---

### Phase 4 ── 収益本格化（目標: 月10万PV・黒字化）
**開放機能**: Google AdSense + プレミアム機能検討

やること:
- Google AdSense申請（HTTPS化後・月1,000PV以上で申請可能）
- A8.netアフィリエイト、ジャンル別商材配置（株・金融 → ネット証券）
- 忍者AdMaxをAdSenseに段階的に切り替え
- 週次まとめ記事の品質向上でSEO流入を増やす
- プレミアム機能案：広告非表示プラン（月250円）

収益目標:
- Phase 4開始時: 広告収入 月1,000円
- Phase 4達成時: 広告収入 月10,000円

---

### Phase 5 ── グローバル展開（目標: 月50万PV）
**開放機能**: 英語版 + API公開

やること:
- 英語RSSフィード追加・英語トピック分類
- flotopic.com/en として英語UIを追加
- Flotopic API（法人向け）の設計・提供
- 収益モデル拡張：法人サブスクリプション
