# ストーリー分岐の判断方針（T2026-0428-BRANCH）

> 制定: 2026-04-29 / オーナー: P003 Flotopic / 関連: T212, T191
> 出典: TASKS.md T2026-0428-BRANCH「注目度（数字）ではなく内容（登場人物・因果関係・エンティティ重複）で分岐判断」

## 1. 背景

ストーリー分岐（親トピック → 子トピックへの分割）は **「ストーリーが本当に枝分かれしたか」を内容で判断すべき**であり、注目度の数値（velocityScore・記事数・トレンド）で分岐させると「同じストーリーが伸びただけのものを別話扱いする」誤判定が出る。

`docs/product-direction.md` 既出方針:
> **注目度軸**（鮮度・グラフ・記事数）と**内容軸**（ストーリー解説・各社見解）を分離する。

本メモはその「内容軸」側の分岐判定方針を、現状コードの読み取り結果を踏まえて言語化する。

## 3. あるべき分岐判断（方針）

### 3.1 原則

> **数字（velocityScore・記事数）は分岐の動機にはならない。**
> 数字は「再処理トリガー（しきい値超えで AI に再評価させる）」までの役割。
> 実際の分岐判断は、以下の 3 つの内容シグナルで決める。

### 3.2 内容シグナル 3 つ

| # | シグナル | 判定方法 | 重み |
|---|---|---|---|
| 1 | **登場人物（主役エンティティ）の重複** | 親トピック主役 ∩ 候補記事主役 = 空集合 → 別ストーリー候補 | 高 |
| 2 | **因果関係の連続性** | 「Aを受けて」「Aの結果」のような因果接続が proc_ai 抽出 timeline 内にあるか | 中 |
| 3 | **エンティティ集合の重複度** | Jaccard(entities_parent, entities_candidate) < 0.3 → 別ストーリー候補 | 中 |

### 3.3 判定マトリクス

| 主役重複 | 因果連続 | エンティティ Jaccard | 判定 |
|---|---|---|---|
| あり | あり | ≥0.3 | **同一ストーリー（マージ）** |
| あり | なし | ≥0.3 | 同一ストーリー（仮説: 続報） |
| あり | あり | <0.3 | **分岐候補**（同人物の別事案） |
| なし | あり | ≥0.3 | 分岐候補（関連事案） |
| なし | あり | <0.3 | **新ストーリー（独立）** |
| なし | なし | <0.3 | **新ストーリー（独立）** |
| なし | なし | ≥0.3 | 関連トピック横リンク（同ジャンル） |

→ velocityScore は出てこない。velocityScore は再処理タイミングの決定にのみ使う。

### 3.4 AI への委任範囲

proc_ai.py のプロンプトに以下を追加する想定:

```
あなたは新しい記事 X が既存トピック T の続報か、別ストーリーかを判定する。
判定材料:
- T の主役（primary_entity）と X の主役
- T の timeline 末尾と X の本文の因果接続有無
- T の entities 集合と X の entities 集合の重複

出力: { "decision": "merge"|"branch"|"new"|"link", "reason": "..." }
```

→ これは **数字を一切渡さない**。AI が内容のみで判定する。
数字（velocityScore）は再処理を起動するか否かのゲートにしか使わない。
