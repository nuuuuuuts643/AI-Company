## 2. 重要な発見

### 発見1: 経済ジャンルの圧倒的なROI
- 蓄積28件（2.2%）で PV 76件（16.6%）— **PV/topic 2.71（2位の2.6倍）**
- AI生成率100%（全件AI処理済）
- avg_articles 7.5（複数メディアが同じトピックを追っている＝重要度の証拠）
- **「経済を集中強化すれば供給を倍にしても需要が捌ける」可能性が高い**

### 発見2: 政治+経済+社会 がコア需要
- 3ジャンルで PV合計 219/458 = **47.8%**
- 蓄積コンテンツ比率は 17.4% にすぎない
- **時事性・速報性が高いジャンルにユーザーが集まっている**

### 発見3: 株・金融の供給過剰
- 蓄積181件（14.1%）に対し PV わずか10件（2.2%）— PV/topic 0.055
- avg_articles 7.8（記事は多い）が velocity_high≥50 はわずか9件
- **「株価ニュースは出回っているが Flotopic で読まれていない」状態。差別化できていない**

### 発見4: 総合ジャンルの薄さ
- 249件（19.4%）と最大量だが PV/topic 0.165
- **「カテゴリ未分類のゴミ箱」になっている可能性。タグ精度改善で他ジャンルへ振り分けるべき**

### 発見5: AI生成率と PV の相関は弱い
- AI生成率高: テクノロジー73.8%, ビジネス71.7%, くらし70.0% → 中位〜下位の PV
- AI生成率低: ファッション15.5%, 健康37.5% → 下位の PV
- **AIで深く解説できるジャンル ≠ ユーザーが見たいジャンル。供給と需要のミスマッチが存在する**

## 3. 集中ジャンル候補（推奨）

### 第一案: 「経済 + 政治」集中
- **根拠**:
  - 実需 PV合計 36% を 17件→ 倍増させれば 50%超を取りに行ける
  - 経済は AI生成率100% で品質コントロール済
  - 政治・経済は時事性が強く「速報性 × ストーリー解説」という Flotopic の AI 4要素構造（状況解説/各社見解/注目ポイント/予想判定）と相性が良い
  - SmartNews・グノシーは網羅性勝負で「個別深堀り」では戦っていない
- **差別化メッセージ**: **「経済・政治ニュースを“なぜ今これが起きているか”まで読み解く AI ニュース」**
- **撤退候補**: 株・金融, グルメ, ファッション, 健康, 文化, くらし, 科学 (合計 PV 30/458 = 6.5%)

### 第二案: 「経済 + 社会」集中
- 第一案より政治色を薄め、生活者に近い社会問題を取り込む
- 社会ジャンルは avg_velocity 14.61, velocity≥50 が12件と最高 — **トレンド性が最も強い**
- 「経済の構造解説 + 社会のトレンド速報」で性質の違うコンテンツを両輪に

### 第三案: 「経済」一点集中
- 最もシャープなブランディング
- daloopa のような institutional-grade financial analysis を消費者向けに簡易化したポジション
- **収益化観点**: 経済ジャンルは金融商品アフィリエイト・Bloomberg/日経級の有料広告と相性が良い（T109 アフィリエイトと連動可能）
- リスク: トラフィックの天井が低く、休刊ジャンル（株・金融）と差別化が難しい

## 4. 推奨アクション

### 短期（今週）
1. **「総合」ジャンルの自動再分類** — 249件のうち他ジャンルに振り分けられるものを ai 分類で再判定（PV/topic 0.165 を底上げ）
2. **株・金融の供給絞り** — fetcher の rss-source で株・金融系を deprioritize。velocity_high≥50 の9件以外は処理しない閾値導入
3. **経済ジャンルの供給増** — 28件しかない。fetcher で経済ジャンル相当の RSS source を追加 + 既存 unknown topic から経済への分類精度を上げる

### 中期（2-3週）
4. **第一案「経済+政治」を3週間試行** — about.html の見出し・OGP の説明文を「経済・政治の深掘りニュース」に書き換え、PV/topic と新規 unique_users がどう動くか SLI で観測
5. **収益計測との接続** — `docs/revenue-log.md` に genre 別 RPM / アフィリエイト CTR を記録し、「経済集中で 1ユーザーあたり収益が上がるか」を測定

### 長期（1-2ヶ月）
6. **集中ジャンル決定後**: ジャンル外コンテンツは段階的に廃止 or 別ドメイン分離（about.html / フッター / sitemap で明示）
7. **AI プロンプト特化** — 経済・政治専用のシステムプロンプトを `proc_ai.py` に分岐（業界用語・人名・組織名の理解度を上げる）

## 5. データの限界・注意点

- **PV絶対数が小さい**: 458件。施策判断には十分でも統計的有意性は低い。1ヶ月分のログ蓄積後に再集計推奨
- **unmatched 216件**: TTL切れ等で削除された topic への過去 PV。ジャンル特定不可
- **bot 排除**: 現在の page_view は bot フィルタ実装の確認が必要（`lambda/api/track.py` 等）
- **home PV 774件**: home からどのジャンルへ遷移したかは現在の sk スキーマでは追跡不可。**遷移ファネル計測の追加実装** が次タスク候補
