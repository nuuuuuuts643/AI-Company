# Flotopic ビジョン・ロードマップ

> 最終更新: 2026-04-27

## ビジョン

**「世界がわかるようになる、数分で」**

ニュースの羅列ではなく、出来事の「文脈の地図」を提供する。縦の因果（大項目→中項目→小項目）と横の波及（ジャンル横断）がつながり、ユーザーが世界の構造を自分ごととして理解できる。

## 設計原則（崩さないこと）

1. **精度が命** — 誤った因果関係は信頼を壊す。確信度低は表示しない
2. **単発記事は出さない** — 記事2件以上・整合性チェック通過のみ
3. **粒度は「具体的な一つの出来事」単位**
4. **表現は断定より示唆**（「可能性がある」「指摘されている」）
5. **ユーザーを主語にする** — 「この出来事が自分の生活にどう影響するか」
6. **実装したら必ずブラウザで確認** — コードだけで完了としない
