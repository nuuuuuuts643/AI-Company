## 1. ゲームコンセプト

「5分で遊べる、奥が深い。」  
通勤・通学のスキマ時間に完結するストラテジー。  
一方で属性チェーンの連鎖・ショップビルド・抽出リスクで長く楽しめる。

### ジャンル
HD-2Dドット絵 × リアルタイム配置 × オートバトル × ローグライト（セッション型）

### ターゲット
- 主要: 25〜35歳男性、RPG/ストラテジー好き、iPhone所持
- 副次: タワーディフェンス経験者、Escape from Tarkov / LoLファン

## 2. セッション構造（ナイトレイン的）

バトルは「1セッション完結型」。ステージを選んで突入し、ウェーブを越えるたびにビルドが強化されていく。セッション終了時にのみアイテムが確定する。

```
[セッション開始]
    │
    ▼
WAVE 1: バトル
    │ クリア
    ▼
ウェーブ間ショップ（LoL型）──── ゴールド獲得 → 強化購入
    │
    ▼
抽出判断（Tarkov型）──── 脱出 or 続行（リスク承知）
    │
    ▼
WAVE 2, 3... → ボス
    │ 全ウェーブクリア
    ▼
[脱出成功 → リザルト → 報酬確定]
```

### セッション特性（ナイトレイン参考）
- セッション内の強化は永続しない（次セッションリセット）
- 毎セッション手札・ショップが異なる → リプレイ性
- プレイ時間は1セッション5〜12分（1ウェーブ90秒ペース）

---

## 3. ウェーブ間ショップ（LoL型ビルド要素）

### 概要
ウェーブクリアごとにゴールドを獲得し、ショップで強化アイテムを購入する。  
`shop_system.dart` の `ShopSystem` クラスが管理する。

### ゴールド計算
```
ゴールド = (ウェーブ番号 × 8 + スコア÷50) × RNG(0.8〜1.2) × ゴールドボーナス倍率
```

### アイテムカテゴリ
| カテゴリ | 例 | 効果 |
|---|---|---|
| ユニット強化 | 戦士の訓練 | 全ユニット攻撃力+15% |
| 属性ブースター | 炎の結晶 | 火属性ダメージ+25% |
| チェーンUP | チェーン触媒 | チェーン発動確率+15% |
| マナ強化 | マナ結晶片 | マナ回復速度+20% |
| 城壁回復 | 城壁修復（中） | 城壁HP+30 即時回復 |
| レア枠（W6+） | チェーン共鳴石 | チェーン倍率+40% |

### ショップ特性
- 表示数: W1〜2は3枠、W3+は4〜5枠（ランダム）
- 再抽選: ゴールド20消費で全ラインナップ刷新
- SessionBuffs: 購入効果はセッション中蓄積、ネストも可能
- スキップ: いつでも「次のウェーブへ」ボタンでスキップ可能

## 4. タルコフ的抽出システム

### アイテム状態
| 状態 | 説明 | 死亡時 |
|---|---|---|
| `atRisk` | バトル中取得。抽出前はロストリスクあり | ロスト |
| `secured` | ボス撃破 or 抽出成功で確定 | 保持 |

### 例外ルール
- **コスメティックアイテム**（`isCosmeticOnly = true`）は死亡してもロストしない
- ウェーブクリア時に5%確率でランダムアイテムが自動secured昇格

### 抽出フロー
```
ウェーブクリア後 →「脱出するか、続行するか」選択画面
    ├─「脱出する」→ 3秒の抽出アニメーション → 全atRiskがsecured → リザルト画面
    └─「続行する」→ 次ウェーブへ（atRiskのまま。リスク継続）
```

### 判断材料として表示する情報
- atRisk アイテム一覧と合計ゴールド価値
- ウェーブ進行バー（残りウェーブ数）
- secured 済みアイテム一覧

## 5. 爽快感設計

### レイヤー構造
| レイヤー | 内容 | 担当コンポーネント |
|---|---|---|
| 即時フィードバック | FloatingText、HapticFeedback | `floating_text.dart` |
| 中期快感 | チェーン演出、部位破壊 | `chain_effect.dart` |
| 長期快感 | ScreenShake（ボス撃破）、FlashWhite | `screen_shake.dart` |
| アニメーション | 行進ボブ・攻撃シーケンス・死亡エフェクト | `animation_controller.dart`（新規） |

### チェーン反応（コア差別化機能）
- 前の攻撃属性 → 次の属性が「連鎖ペア」のとき発動
- ×2: 青光波紋、×3: 紫光、MAX: 金色フラッシュ + ScreenShake
- 確率要素: 装備・ショップバフで70〜100%

## 6. 属性システム（じゃんけん型）

```
   火 → 風 → 土 → 火   （正三角形A: 基本三すくみ）
   水 → 火            （水は火に強い）
   光 ↔ 闇           （光闇互いに弱点）
   土 → 水            （大地が水を吸収）
   風 → 光            （疾風が光を分散）
   闇 → 土            （闇が大地を侵食）
```

- **弱点**: ×1.5 + 黄色FloatingText + HapticFeedback（軽）
- **耐性**: ×0.6 + グレーFloatingText
- **通常**: ×1.0

## 7. 運の要素 6種

| # | 要素 | 確率帯 | 影響 |
|---|---|---|---|
| 1 | カードドロー | ランダム | 戦術を毎回変える（高影響） |

| 4 | ウェーブ内敵数 | ±20% RNG変動 | 毎回違う緊張感 |
| 5 | ショップ抽選 | プールから RNG | ビルドが毎回異なる |
| 6 | チェーン発動確率 | 70〜100%（バフ依存） | 計算できるが保証なし |
